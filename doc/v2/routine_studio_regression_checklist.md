# Routine Studio 回帰チェックリスト

本チェックは `routine studio` のリファクタリング後に最低限確認する項目。
`RSR-010` の受け入れ確認として利用する。

## 1. 事前条件
- `npm run typecheck` が成功していること
- モジュール一覧を取得できる状態であること（APIモック/実環境どちらでも可）

## 2. 編集フロー
1. ルーティン画面を開き、ライブラリにモジュールが表示される
2. `追加` ボタンでモジュールをキャンバスに挿入できる
3. `×` ボタンでエントリ削除できる
4. `↑/↓` で並び替えできる
5. カードのハンドルドラッグで並び替えできる
6. エントリ設定モーダルで `タイトル/分/モジュール/ノート` が更新される

## 3. 履歴フロー
1. キャンバス編集後に `元に戻す` が有効化される
2. `元に戻す` で直前状態に戻る
3. `やり直す` で再適用できる
4. 履歴移動後も選択エントリが不正状態にならない

## 4. モジュール管理フロー
1. `モジュールを追加` で新規モーダルが開く
2. 保存後、ライブラリに反映される
3. 既存モジュールの編集が反映される
4. 削除後、ライブラリから消える

## 5. テンプレートフロー
1. `保存` でテンプレート保存が成功する
2. 既存テンプレート保存時は更新扱いになる
3. 複合モジュールの `読込` がキャンバス置換として動作する
4. 複合モジュールの削除が反映される

## 6. 今日適用フロー
1. `今日に適用` 実行でテンプレート保存後に適用される
2. 適用結果メッセージ（Applied/Shifted）が表示される
3. 適用後に再描画しても状態が崩れない

## 7. 失敗時確認
- モジュール削除対象が存在しない場合、`module not found` が表示される
- レシピ削除対象が存在しない場合、`recipe not found` が表示される
